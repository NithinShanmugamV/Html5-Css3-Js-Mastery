function StudentMarks(physics, chemistry, maths){
this.physics = physics;
this.chemistry = chemistry;
this.maths = maths;
this.greeting = function(){
console.log('hi');
};
this.aggregate = function() {
return (this.physics/2)+(this.chemistry/2)+this.maths;
};
}
let stu1 = new StudentMarks(75,80,82);
console.log(stu1.greeting());
console.log(stu1.aggregate());
Instead of returning an object, we're going to use a different approach to initialize an object. In JavaScript, we have a keyword called this . And this is a reference to the object that is executing this piece of code.

For now, just imagine this references an empty object. Now you know with dot notation, we can access properties of an object, we can read a property, or we can set a property.
So on this new empty object, we want to add a property called physics, and we set that to this physics argument that you receive as parameter. Similarly do this for chemistry and maths. So in JavaScript our objects are dynamic. Once we create them we can always add additional properties or methods to them. So here we are adding a new property but empty object.
when we use new operator here, 3 things happen.
  • This new operator creates an empty JavaScript object. Something like this -> let stu1 ={};
  • Next, it will set this keyword to point to this new empty object stu1. So in this code we have access to this empty object(stu1), and we set all the properties that uses this keyword to point object stu1.
  • Finally, this new operator will return this new object from this function. Previously we added return in factory functions but new operator will take care about this.
